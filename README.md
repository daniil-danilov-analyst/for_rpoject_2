# Анализ пролонгаций договоров (2023)

## Описание проекта

Проект посвящён расчёту ключевых метрик эффективности аккаунт-менеджеров по пролонгации договоров с клиентами. На основе данных о завершении проектов и финансовых отгрузках  рассчитаны два типа коэффициентов пролонгации:

- **К1** – доля выручки, пролонгированная в первый месяц после завершения проекта.
- **К2** – доля выручки, пролонгированная во второй месяц (для проектов, не пролонгированных в первый).

## Описание данных

В проекте используются два набора данных:

### 1. `prolongations.csv`

| Поле | Описание |
|------|----------|
| `id` | ID проекта |
| `month` | Последний месяц реализации проекта |
| `AM` | ФИО ответственного аккаунт-менеджера (данные первичны по отношению к `financial_data`) |

### 2. `financial_data.csv`

| Поле | Описание |
|------|----------|
| `id` | ID проекта |
| `Причина дубля` | Причина, почему строки с одним и тем же `id` встречаются несколько раз |
| `Колонки с названием месяца` | Сумма отгрузки проекта в данный месяц (например, `Ноябрь 2022`, `Декабрь 2022`, …) |
| `Account` | ФИО ответственного аккаунт-менеджера |

#### Специальные значения в колонках месяцев

В ячейках с отгрузками могут встречаться следующие значения:

| Значение | Описание |
|----------|----------|
| `в ноль` | Отгрузка проекта в данном месяце равна 0. Для расчёта коэффициента пролонгации берётся отгрузка предыдущего месяца (только если все части оплаты равны 0). |
| `стоп` | Проект закончился до истечения срока договора. Если у проекта есть “стоп” в последний месяц реализации или ранее, такой проект исключается из расчёта пролонгаций. |
| `end` | Аналогично `стоп` – проект завершён досрочно, исключается из анализа. |
  
## Основные этапы анализа

1. **Предобработка данных**
   - Очистка от "стоп"/"end" (проекты исключаются из расчёта)
   - Замена "в ноль" на 0, преобразование формата чисел
   - Разделение транзакций на «основные» и «дополнительные» (вторая часть оплаты, доп работы и т.д.)
   - Приведение данных к длинному формату для удобства расчётов

2. **Расчёт коэффициентов пролонгации**
   - Ищем пролонгированные проекты 
   - Для каждого проекта, определяется сумма отгрузки в последний месяц реализации (до пролонгации)
   - Находятся отгрузки в следующем (первом) месяце → К1; если их нет – ищутся во втором месяце → К2
   - Вычисляются индивидуальные коэффициенты для каждого события пролонгации

4. **Агрегация и визуализация**
   - Помесячные и годовые коэффициенты по каждому менеджеру
   - Сводная статистика по отделу
   - Графики динамики коэффициентов во времени
